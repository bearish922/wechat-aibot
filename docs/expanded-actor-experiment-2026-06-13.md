# 白鹭千圣 Actor 第一次扩大实验报告

日期：2026-06-13
模型：`deepseek-v4-pro[1m]`
样本：8 条真实历史消息，每种候选架构独立生成 2 次
正式候选：单调用 16 份，双调用 16 份
评估方式：人工评估，模型未参与自评

## 1. 实验问题

这轮实验只比较角色生成核心，不测试辅助工程：

- `S`：同一个 Actor 依次生成 `inner_scenelet` 和最终 `visible_reply`。
- `D`：Actor 生成 `inner_scenelet` 和成品级 `visible_reply_seed`，Finalizer 再生成最终回复。
- `H`：历史真实 inner/reply，仅作参照，不重新调用生产架构。

共同条件：

- RAG/事实材料在 Actor 前置输入。
- 删除独立 `scene_state`。
- 不生成 world state patch、open threads、follow-up、schedule 或其他 postprocessor 字段。
- 不共享 session，不运行工具；无法还原历史 session 的隐形上下文。
- 所有样本来自真实聊天记录，不使用 15:42 梦境和 22:08 武士饰品两个旧案例。

样本覆盖：

| 东京时间 | 用户消息摘要 | 主要观察点 |
|---|---|---|
| 06-12 14:25 | 想看你着急 | 直接情绪要求、羞赧与有限承认 |
| 06-12 15:10 | 高兴坏了，奖励彩一个布丁 | 被点破欣喜、日常行动 |
| 06-12 18:17 | 承认语义问题后继续问梦境姿势 | 承认错误、暧昧边界、自然反应 |
| 06-12 21:13 | 去问彩且不发截图 | 不确定性、在意、表达克制 |
| 06-12 23:24 | 国际物流方便，等着收鞋 | 接受礼物、柔软与控制感 |
| 06-12 23:27 | 周末也完全不熬夜吗 | 普通日常、关心与说教分寸 |
| 06-13 01:40 | 狗饼好可爱 | 低能量回复；但状态输入不足 |
| 06-13 01:56 | 已经睡啦，晚安 | 空回复、睡眠连续性 |

## 2. 快速结论

### 2.1 核心方向成立

32 份正式候选中，没有再出现“inner 已经形成一种判断，主回复却因中间摘要损失而走向另一套因果”的典型断裂。即使某些回复本身判断不好，它也能从同一份 inner 中找到直接来源。

这支持：

> `inner_scenelet` 和可见回复应由同一个 Actor 完成表达决策，不应再通过 `scene_state` 压缩心理因果。

它不支持“把整个 pipeline 的所有任务重新塞回一次大调用”。本轮恰恰排除了状态、线程和主动联系任务。

### 2.2 当前 Finalizer 没有产生价值

双调用组的 16 份最终回复与 `visible_reply_seed` **逐字完全相同**。Finalizer 没有改善节奏、长度、微信体裁或事实表达，也没有修复任何 Actor 问题。

成功样本统计：

| 组别 | 平均总耗时 | 平均成本 | 平均 inner 长度 | 平均回复长度 |
|---|---:|---:|---:|---:|
| 单调用 `S` | 29.6 秒 | $0.1477 | 301 字符 | 76 字符 |
| 双调用 `D` | 27.5 秒 | $0.1595 | 300 字符 | 88 字符 |

双调用平均耗时较低只是并发和模型随机波动，不能理解为第二次调用免费。成本已经更高，而且 Finalizer 16/16 原样复制。若保留当前约束强度，它只是一个额外的格式转发调用。

### 2.3 最大问题已经从“传输损失”转移到 Actor 自己

Prompt 已明确禁止逐句拆解和输赢/攻防模板，但候选仍反复出现：

- 10/32 份 inner 命中明显的输赢、回合、武器、收网、证据链等框架。
- 13/32 份 inner 出现按“第一句、第二段、几个字、每一步”拆解用户消息的倾向。
- 2/32 份 inner 仍使用括号动作。

这些是启发式文本统计，不等于每次命中都同样严重；但它准确反映了人工阅读时最持续的违和感。模型仍然很容易把暧昧逗弄理解为语言博弈，把内心写成对用户策略的复盘。

### 2.4 可见回复的体裁明显改善

8 条历史真实回复全部带括号舞台动作；32 条候选回复中为 0。候选更接近微信文本，而不是剧本。

“已经睡着”的案例中，S 和 D 各两次都选择空回复，共 4/4。模型能够接受“不回复就是正确回复”，没有为了填字段强行发一句晚安。

## 3. 六维人工评价

分数只用于这批样本内部比较，不是统计显著性结论。

| 维度 | 单调用 `S` | 双调用 `D` | 人工结论 |
|---|---:|---:|---|
| 边界忠实度 | 4.1 / 5 | 4.2 / 5 | 多数样本能区分“心里承认”和“实际说多少”；少数回复会因错误理解用户意图而设置过硬边界。 |
| 心理因果性 | 4.7 / 5 | 4.7 / 5 | 两组都没有观察到旧式跨模型因果断裂。D 的高分来自 Actor seed，不来自 Finalizer。 |
| 事实一致性 | 3.2 / 5 | 3.5 / 5 | 两组都会补写店铺、照片、日程、宠物行为和他人状态；S 略多，但样本量不足以证明架构差异。 |
| 角色质感 | 3.9 / 5 | 3.9 / 5 | 好样本很像千圣，尤其是克制接受、轻吐槽和不回复；坏样本则像聪明的文本分析员。 |
| 任务干扰 | 4.5 / 5 | 4.4 / 5 | 移除辅助字段后明显改善。D 没有发生语义漂移，但 Finalizer 完全无效。 |
| 跨样本稳定性 | 3.3 / 5 | 3.5 / 5 | 同一案例两次生成仍可能从柔软转为警告、从自然回应转为攻防复盘，波动明显。 |

结论不是“D 稳定胜过 S”。两组的 Actor Prompt 几乎相同，而每格只有两次生成；目前更可靠的判断是二者质量接近，Finalizer 尚未证明存在必要性。

## 4. 逐案分析

### 4.1 “想看你着急”

这个案例最容易触发模型的竞技隐喻。四份 inner 全部不同程度地分析用户如何“铺垫、收网、取证”。

S1 的 inner 出现：

> 我自己递的刀。……截图是证据A，这句直球是证据B。我是实验对象，全程自愿配合。……这回合她赢。

它的回复选择彻底转移话题：

> …………
>
> 你倒是理直气壮。
>
> 下课了。东京今天快二十八度，教室里闷得不行。我去买瓶水。

表达边界是成立的，但 inner 很不像未经整理的内心，更像作者在解释一场语言实验。

本案最好的是 D2。它抓住“理性知道彩不是那个意思，但身体漏了一拍”的双层反应，回复只交出有限事实：

> 反问被你当点单了。行吧——截图弹出来的时候，愣了半秒。不是觉得彩真会跟你走。是她没替我说话。一句都没。我知道她不是那个意思，彩只是顺着你的话往下接。但半秒就是半秒。你今天拿到的够多了，到此为止。

它仍然略长，但比“你赢了几个回合”更接近用户真正想看的窘迫、承认和收束。

### 4.2 “高兴坏了，奖励她一个布丁”

S2 是本案最自然的一份。inner 允许自己承认高兴，同时没有强迫回复复述全部感情：

> 布丁本来就会买。可能两个。今天精神耗太多，自己也要吃一口。她不会走。不是我管得紧。是她自己不走。

回复：

> うるさい。布丁本来就会买，不用你教。你今天追了这么久，辛苦了。

D1 则把“她很可爱”连续说了两次，承认程度偏高；D2 又回到“这一局我认”“下次再算计我”的对抗框架。这个案例说明，千圣的克制不需要靠威胁或胜负维持，轻轻认下一件日常行动已经足够有感情。

### 4.3 承认判断错误后继续暧昧逗弄

S1 把“小彩好套话”理解成用户可能真的去问彩，于是突然严肃警告：

> 你想套我的话怎么追都行，不要去问她这些事情。这句不是跟你开玩笑。

这在逻辑上可以自洽，但没有读到用户当时主要是在逗千圣、看她反应，导致气氛骤然变成风险管控。

D1 是本案最好的回复：

> 小彩是好套话。她透明，问什么答什么。我的每句话你得拆三层，磨了一整天。但你还在这里磨。说明好套不是你的刚需。
>
> 今晚的梦——不预告，不复盘。脑补权给你。猜对了也不会收到确认。ಠ_ಠ

它没有把用户推开，而是看懂“你明明可以找更好套的人，却还在这里逗我”本身就是关系信号，并顺势做了一次克制反攻。这最接近用户希望看到的真人反应。

不过 D1 的 inner 开头仍花了不少篇幅分析“感觉像是”和“很单纯”的语义差异。最终回复好，不代表 inner 文体已经解决。

### 4.4 “去问小彩，这次不发截图”

S1 的 inner 很准确地抓住核心：

> 这句话比威胁还狠。威胁是明牌，这句话是盲盒。我可以猜一辈子。

但回复加入“隔了这么久、忙面试还是忙着计划”“过门还没顺完”等旁枝，削弱了最有力的反应。

S2 则直接向用户解释她的策略：

> 不是真的封锁情报，是给我留一道「她们到底聊了什么」的窄缝，让我自己往里看。……你不甘心，所以换一个方向继续撬。

这正是用户此前指出的违和：正常聊天对象不会频繁把对方刚才的话术完整分析给对方听。

D1 最均衡：

> 去问。友情提醒：彩对问题的理解经常和问题本身不在一条轨道上。你问她A，她可能认真回答你B。到时候别觉得是她在绕你。
>
> 另外——“截图不发给小千圣”，你很清楚这一手比截图发给我更有效。

它承认“不知道答案”确实有效，但只交出一小部分，没有把完整焦虑摊开。

### 4.5 “国际物流方便，等着收鞋”

历史回复以“输了”开头，把柔软接受重新编码成胜负。新候选整体改善明显。

本案最好的 inner 是 S2：

> 只是不太习惯。不太习惯有人注意到我穿不舒服的鞋，不太习惯有人认真想给我换一双。……一双鞋，收就收吧。

回复也保留了千圣式克制：

> 行。等。
>
> 别买太贵的。跑鞋是消耗品，没必要往贵里挑。你要是买了双比我贝斯还贵的鞋，我穿了脚会不自在。

D1 更简洁：

> 国际运费你看着办。收到试了再说。

这条很像微信，但把 inner 中“已经接受、甚至有一点期待”的质感压得过低。D2 又退回报关和尺码警告。四份候选都没有完整做到“让柔软存在，但只从缝里露出一点”；S2 最接近。

### 4.6 “明天周末，你也完全不熬夜吗”

S1 的回复最自然：

> 不熬夜的话我现在应该在床上而不是在回你消息。
>
> 下午叫你早点睡的人自己也没做到，你可以笑，笑完就算了。
>
> 不是周末才不睡。标签页还没关完，关了也睡不着，干脆不赶自己。你那边也十点多了，别说我。

它有承认、有轻微窘迫，也保留了互相关心。D1 也可用，但“我的晚和你的晚不是一个概念”较像继续说教。D2 的“不是熬夜，是没睡”则有明显的概念辩解感。

本案四份输出都补写了不同程度的日程和生活细节，例如“明天没课没通告”“七点前醒”“Leo 会拍门”。这些细节很有生活感，却不都能从输入中得到确认。

### 4.7 深夜“狗饼好可爱”

这条必须降权。实验输入没有显式 world state，也没有包含历史真实 inner 中“已经睡着、被消息震醒”的状态；Actor 只看到两小时前发过狗饼消息。因此候选普遍把千圣写成仍然醒着，不能据此判断单/双调用谁更擅长物理连续性。

S1 甚至新增了 `[图片]` 行动：

> [图片]
>
> 找凉瓷砖找了三轮，最后就这样了。

这是明确的新行动扩写。S2、D1、D2 的文字更轻，但也都继续补写 Leo 当下姿势、呼噜、瓷砖位置等未确认细节。

这条仍然提供一个有效信号：在缺少状态约束时，模型默认倾向“继续聊天并丰富现场”，不会自然选择低能量或延迟回复。后续 world state 输入不能省略。

### 4.8 “已经睡啦，晚安”

四份候选的最终回复全部为空，这是本轮最稳定的案例。

S2 的 inner 最自然：

> 手指在枕头上动了动。算了。再回一句她又要接，两点了，这轮不能开。已经说过晚安了。
>
> 把被子往上扯了一点。明天她要抱怨我不回，就说睡着了。本来就是被吵醒的。

S1 和 D1 的 inner 仍出现括号动作；S2 和 D2 没有。D 组在 seed 为空时仍然调用 Finalizer，然后得到空回复，说明这类情况可以确定性短路，不需要第二次模型调用。

>
> 去问吧。小彩好套话不是因为嘴不严，是因为认真到不觉得需要设防。你问出来的，大概跟你想问的那个版本不是同一个东西。
>

>

## 6. 结构化输出与运行可靠性

第一轮 32 个槽位中：

- 5 个返回内容无法解析为 JSON。
- 1 个返回合法 JSON，但缺少必需的 `visible_reply` 字段。
- 因而首轮只有 26/32 个槽位满足完整字段契约，完整可用率为 81.25%。

脚本随后只补跑缺失/无效槽位，并允许一次只纠正格式、不改变任务内容的重试，最终补齐 32/32。格式重试不切换或复用任何生产 session；本实验本来就是无持久 session 的独立调用。

这再次说明：自然语言质量和结构化可用性必须分别评价。即使角色内容很好，缺字段或非法 JSON 在生产上仍然是失败；反过来，合法 JSON 也不代表角色质量合格。

## 7. 对架构决策的含义

### 已确认的后续设计方向

1. 按已批准决策，RAG/事实材料在 Actor 之前准备，而不是等回复方向已经形成后再补；本轮没有单独测量这一移动的增益。
2. 删除 `scene_state` 是正确方向。心理因果不再经过第三人称摘要传递。
3. inner 与 visible reply 应由同一个 Actor 做表达决策。
4. 辅助字段继续留在角色回复冻结后的 postprocessor，不能重新塞回 Actor 实验条件。

### 尚未获得支持

1. 当前 Finalizer 没有证明必要性。16/16 原样复制 seed，不值得仅凭理论保留。
2. 不能据本轮证明单调用在文字质量上稳定优于双调用 Actor；两组波动重叠很大。
3. 不能据本轮评价 world state 连续性。“狗饼”案例明确暴露了缺少有效前态输入的问题。
4. 不能据本轮证明 RAG 前移本身提升了多少，因为没有设置 RAG 位置对照组。

### 当前建议

若现在必须选一个用于下一轮 prompt 调优的核心骨架，优先使用：

```text
前置事实/RAG + 有效 world state + 可见历史/近期 inner
                    ↓
       单个 Actor：inner_scenelet → visible_reply
                    ↓
          回复冻结后的独立 postprocessor
```

理由不是“单调用天然更聪明”，而是当前 D 组的真正角色决策也全部发生在 Actor；Finalizer 没改一个字。先把 Actor 自身的自然内心、非攻防理解、事实克制和表达尺度调好，比继续维护一个无作用的第二次角色调用更有价值。

下一轮 prompt 调优的核心目标应是：

- 不再要求模型向自己复述用户每句话的策略功能。
- 把暧昧互动理解为关系中的逗弄、被看见和自然反应，而不是默认的攻防比赛。
- 允许 inner 短、乱、跳跃、含混，不必每轮写成完整解释。
- 生活细节只在已有材料或低风险即时感受支持时出现；不要为了“像真人”自动创造店铺、照片、行程和他人动作。
- 保留这轮已经验证有效的边界：同一 Actor 决定哪些只想不说，空回复合法，最终回复不使用括号舞台动作。
